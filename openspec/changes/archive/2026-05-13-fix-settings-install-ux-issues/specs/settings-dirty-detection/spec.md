## MODIFIED Requirements

### Requirement: 设置页未保存改动检测
设置页 SHALL 只在用户实际修改了设置后才提示"有未保存的改动"，在数据加载完成前不触发脏检测。

#### Scenario: 首次进入设置页不应提示未保存改动
- **WHEN** 用户首次打开设置页面，数据尚未加载完成
- **THEN** 不显示"有未保存的改动"提示，脏检测返回 false

#### Scenario: 数据加载完成后无修改不应提示
- **WHEN** 设置数据加载完成，用户未做任何修改
- **THEN** 不显示"有未保存的改动"提示

#### Scenario: 用户修改设置后应提示
- **WHEN** 设置数据加载完成，用户修改了任一设置项
- **THEN** 显示"有未保存的改动"提示
