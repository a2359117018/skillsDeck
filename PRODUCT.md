# Product

## Register

product

## Users

Developers who work with AI coding agents (Claude Code, Cursor, Codex, Windsurf, and 50+ others). They're technical, opinionated, and efficient. They use this tool between coding sessions to search, install, update, and remove skills across multiple agents simultaneously. Their context: a desktop app open alongside their IDE and terminal, glancing at it briefly to manage skills before getting back to code.

## Product Purpose

A desktop GUI for the `npx skills` CLI that makes managing AI agent skills visible and actionable. Success means: a developer never needs to open a terminal to install, update, or remove a skill. The multi-agent complexity (50+ agents with different skill paths) is hidden behind a clear, fast interface. The search API at skills.sh is the discovery layer; this app is the management layer.

## Brand Personality

Confident, powerful, decisive. Three words: sharp, intentional, unapologetic.

The interface should feel like it was built by someone who uses it daily: every element earns its place, every interaction is fast, nothing is decorative. It should give developers a sense of control and mastery over their multi-agent setup, the way Linear makes project management feel precise or Figma makes design feel fluid.

Not friendly. Not warm. Not trying to be liked. Trying to be trusted.

## Anti-references

- Generic SaaS dashboards with blue-white color schemes, metric cards, and template grids
- "Modern" UI kits that look like every other Tailwind starter
- Overly animated interfaces that prioritize delight over speed
- Admin panels that feel like they were generated, not designed

## Design Principles

1. **Expert confidence.** Every design choice should feel decisive. If a button exists, it's exactly the right shape and size. If a color is used, it's there for a reason. No hedging, no "play it safe" compromises.

2. **Tool disappears into the task.** The best feedback is no feedback. Install a skill, see it appear. Remove it, see it gone. The UI should feel like a natural extension of the developer's mental model, not a separate application they have to learn.

3. **Multi-agent clarity.** This app's unique value is managing skills across 50+ agents. The interface must make this complexity feel simple: clear agent association, obvious install targets, no ambiguity about where a skill lives.

4. **CLI power, GUI clarity.** Every action maps to a real `npx skills` command. The GUI should expose that power, not simplify it away. Advanced users should never feel limited by the interface.

5. **Speed over polish.** A developer's time between tasks is measured in seconds. Interactions should be instant, feedback immediate, navigation direct. No loading theatrics.

## Accessibility & Inclusion

Target WCAG 2.1 AA compliance as a baseline. Primary users are developers on desktop (Windows, macOS, Linux) with standard monitors. Support keyboard navigation throughout. Ensure sufficient color contrast for all text. Respects prefers-reduced-motion where applicable.
