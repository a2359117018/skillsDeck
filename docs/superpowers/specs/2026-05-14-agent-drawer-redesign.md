# Agent 技能抽屉 UI 重设计

## 背景

Agent 管理页面中，点击 Agent 卡片打开的技能抽屉存在三个体验问题：

1. 头部"打开文件夹"按钮与 Agent 名字区域间距过大，不够紧凑
2. 技能列表的"更新"/"删除"文字多余，应改为纯图标 + tooltip
3. 技能列表样式过于单调（内容少时更明显）

## 设计方案

融合三版视觉原型的最佳元素：A 单列布局 + B 极光蓝配色 + C 序号标签 + 红色删除按钮。

### 头部（紧凑化）

- 头像 + 名字/计数徽章 + 文件夹按钮/关闭按钮全部在同一行 `flex` 排列
- 头像使用蓝→青渐变背景（`linear-gradient(135deg, #2563eb, #06b6d4)`）
- 技能计数用蓝色徽章显示（背景 `rgba(37,99,235,0.08)`，文字 `#475569`，圆角 9999px）
- "打开文件夹"和"关闭"改为 34×34 图标按钮，无文字，hover 时蓝色背景 + tooltip
- 整体高度约 70px（原方案约 90px），更紧凑

### 技能列表（卡片化）

- 列表区域背景 `#eff6ff`（极光蓝浅色），卡片背景白色
- 每个技能是一张独立圆角卡片（border-radius: 12px，border: 1px solid rgba(37,99,235,0.10)）
- 卡片内容：左侧序号标签 + 技能名，右侧操作按钮
- 序号标签：28×28 蓝色背景方块（背景 `rgba(37,99,235,0.08)`，文字 `#2563eb`），显示 `01`-`N`
- hover 效果：边框变 `#2563eb`，加 `0 2px 12px rgba(37,99,235,0.1)` 阴影
- 卡片间距 8px

### 操作按钮（图标化）

- 纯图标按钮（无文字），尺寸 32×32，圆角 8px
- 默认状态：灰色图标（`#475569`）
- 更新按钮 hover：蓝色底色（`rgba(37,99,235,0.08)`）+ 蓝色图标（`#2563eb`）
- 删除按钮 hover：红色底色（`#fef2f2`）+ 红色图标（`#ef4444`），与更新形成强对比
- 两个按钮均带 tooltip（hover 时显示"更新"/"删除"）

## 涉及文件

仅修改一个文件：`src/renderer/src/views/AgentView.vue`

改动范围：

- `<template>` 中 drawer-header 和 skill-row 部分
- `<style scoped>` 中对应的 CSS

## 不做的事

- 不修改 Agent 卡片网格的样式
- 不修改 drawer 宽度（保持 500px）
- 不新增组件文件，所有改动在 AgentView.vue 内完成
- 不引入新的依赖
