# 架构图绘制指南

## 核心规范

- **必须用 SVG 绘制**，不要用 HTML div 模拟框图
- **箭头用 `<marker>` 定义**，不要用文字符号（→、=>）
- **圆角矩形**：`rx="12" ry="12"`
- **边框**：`stroke-width="2"`
- **字体**：`font-family="system-ui, sans-serif"`

## 颜色编码

| 层级           | 背景色    | 边框色    | 文字色    |
| -------------- | --------- | --------- | --------- |
| 表现层 (UI)    | `#EFF6FF` | `#3B82F6` | `#1E40AF` |
| 逻辑层 (Logic) | `#FEF3C7` | `#F59E0B` | `#92400E` |
| 数据层 (Data)  | `#D1FAE5` | `#10B981` | `#065F46` |
| 外部服务       | `#F3E8FF` | `#8B5CF6` | `#5B21B6` |
| 用户/客户端    | `#FFF7ED` | `#F97316` | `#9A3412` |

## marker 定义示例

```svg
<defs>
  <marker id="arrow" markerWidth="10" markerHeight="7" refX="9" refY="3.5" orient="auto">
    <polygon points="0 0, 10 3.5, 0 7" fill="#6B7280"/>
  </marker>
</defs>
```

## 好的架构图特征

- 每个模块用圆角矩形，内部有标题 + 一行描述
- 层级用颜色区分，同一层级的模块颜色一致
- 箭头清晰，方向明确，用 `<marker>` 定义箭头而非文字
- 底部有图例说明

## 差的架构图（避免）

- 只有文字没有图形 — 那不是图，是列表
- 箭头用 `→` 或 `=>` 文字模拟
- 颜色混乱，没有统一体系
- 模块大小不一致，对齐混乱
- 没有层级的视觉区分

## 实心 vs 虚线

- 实线：直接引用、同步调用
- 虚线：异步依赖、可选模块、外部服务
