空状态（未解析 URL 时）的布局对比，目标：页面不空荡、流程自然、与压缩包安装清晰区分

方案 A — 镜像压缩包布局

与压缩包安装完全一致的网格结构，仅通过左侧输入区域的内容类型区分（URL 输入 vs 拖拽区）

优点
  • 与压缩包安装结构一致，学习成本为零
  • URL 输入区占满左列宽度，视觉充实
  • 步骤编号清晰引导流程
缺点
  • URL 输入区与拖拽区形状相似但功能不同，可能混淆
  • 区分度依赖内容（链接输入框 vs 文件拖拽），视觉差异较小

方案 B — 顶部输入栏 + 双列内容

URL 输入独立在顶部全宽展示，下方双列始终可见。解析前左侧显示仓库信息占位，右侧显示目标选择。

优点
  • URL 输入始终全宽可见，不被挤压
  • 输入栏与内容区分离，结构清晰
  • 与压缩包安装的拖拽区形成明显的视觉差异
缺点
  • URL 输入栏占顶部空间，双列可用高度减少
  • 步骤 1 和 2 的概念被弱化（无步骤编号）
  • 与压缩包安装布局差异较大，一致性降低
