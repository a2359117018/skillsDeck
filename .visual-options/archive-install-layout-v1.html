三种独立 UI 方案，不再共用 LocalInstallPanel

方案 A：上下分栏 + 紧凑横排

优点

  • 空间利用率最高，所有内容紧凑排列
  • 拖拽区 + 技能标签 + 目标标签一目了然
  • 按钮始终可见，不会被截断

局限

  • 技能数量多时横排标签会换行，需要折行处理
  • 无法展示技能的详细路径信息
  • Agent 列表用 "+N 更多" 折叠，展开需要额外交互

适合：技能数量少（1-5个）的快速安装场景

优点

  • 拖拽区移到左栏顶部，紧凑不占纵向空间
  • 左右两栏独立滚动，互不影响
  • 步骤编号引导清晰，操作流程直观
  • 安装按钮固定在右栏底部，始终可见

局限

  • 左右分栏在小窗口下可能显得拥挤
  • 拖拽区尺寸较小，交互感不如全宽拖拽区

适合：需要展示技能详情和完整 Agent 列表的场景

优点

  • 保留全宽拖拽区的良好交互体验
  • 技能以横排卡片展示，紧凑直观
  • 安装目标可折叠，按需展开节省空间
  • 安装按钮固定底部，始终可见

局限

  • 折叠面板增加了一次额外点击交互
  • 上下排列的 Agent 列表在折叠面板内空间有限
  • 技能多时横排卡片需要横向滚动

适合：保留原有拖拽体验，同时解决空间不足的问题

推荐方案
方案 B（左右分栏）最符合你描述的需求——拖拽区紧凑嵌入左栏顶部，左右两栏各自独立滚动， 安装按钮固定在右栏底部始终可见。技能列表和 Agent 列表都能完整展示，不会被截断。 相比方案 A 的标签式，B 可以展示技能路径等详细信息；相比方案 C 的折叠式，B 不需要额外点击操作。
