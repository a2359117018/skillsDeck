GitHub 安装完整调度流程
① 用户粘贴 GitHub URL
例如: https://github.com/owner/repo/tree/main/skills
 UI
② 点击「解析」按钮
GitHubInstaller.vue 调用 window.api.skills.parseGitHub(url)
UI
③ IPC: skills:parse-github
preload → main process 路由到 skills.ipc.ts handler
 IPC
④ GitHubSkillInstaller 解析 URL
提取 owner / repo / branch / subPath，拼接 zipball URL
 Service
⑤ fetch 下载 zipball
AbortController + 30s 超时，通过 IPC 流式上报下载进度
 Network
⑥ 解压到临时目录
decompress 解压，subPath 存在时截取对应子目录
 FS
⑦ LocalSkillInstaller.scanSkills(dir, maxDepth=2)
递归扫描 SKILL.md，遇到即停止深入该分支，返回 skill 列表
 Service
⑧ 返回扫描结果到 Renderer
skill[] 数组：{ name, path, relativePath } 在 SkillScanResult 中展示
IPC
⑨ 用户选择要安装的 skill + 目标 agent
SkillScanResult 复选框 + AgentSelector 多选 → LocalInstallPanel
UI
⑩ 点击「安装」
调用 window.api.skills.installLocal(skillDirs, agents)
 UI
⑪ IPC: skills:install-local
main process 接收 skill 目录列表和目标 agent 列表
 IPC
⑫ LocalSkillInstaller.installSkills(skillDirs, agents)
对每个 skill 目录，用 fs.promises.cp 复制到每个选中的 agent global skills 目录
Service
⑬a 成功 → 清理临时目录
fs.promises.rm(tempDir, { recursive: true })
 FS
⑭a 返回安装结果 + 刷新技能列表
Pinia store 调用 skillsStore.refresh()，InstalledList 自动更新
UI
⑬b 失败 → 同样清理临时目录
无论成功失败，finally 块中清理，避免磁盘泄漏
 Error
⑭b 返回错误详情 → UI 提示
CommandErrorInfo 序列化后展示在对应 Tab 内
 Error
压缩包安装流程（与 GitHub 对比）

与 GitHub 流程的区别用高亮标注

↓ 之后与 GitHub 安装第⑦步合并 ↓
LocalSkillInstaller.scanSkills
→
展示复选列表
→
选择 agent
→
安装 → 清理 → 刷新
关键差异点
  • 无需网络下载（步骤⑤跳过），直接从本地文件开始
  • ArchiveSkillInstaller 替代 GitHubSkillInstaller 做解压
  • 解压后复用同一套 scanSkills + installSkills 逻辑
  • 不涉及 AbortController / 下载进度，流程更简单
